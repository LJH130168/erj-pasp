# 接口说明

目前支持渔翁和得安两家加密机厂商，具体使用根据用户注册应用申请的对应加密机厂商。在用户购买对应的api产品后，会在 应用管理>应用令牌管理 为用户生成对应应用的令牌和 密钥，作为api调用时头部必要参数，用户需妥善保管。支持用户实时更换应用 密钥，更换后需要替换之前您本地的 令牌和 密钥，同时之前的作废。接口有 最大使用期限和 最大使用次数限制，在限制范围内使用有效。

# 签名生成

① 获取baseString

	baseString由令牌和当前时间戳ts拼接而成；
	假如Authorization = 595f23df，CurrentTime = 1512041814，则baseString = 595f23df1512041814
1
2

② 对baseString进行MD5

	假如baseString为上一步生成的595f23df1512041814，MD5之后则为 0829d4012497c14a30e7e72aeebe565e
1

③ 以secretkey为key对MD5之后的baseString进行HmacSHA1加密，然后再对加密后的字符串进行base64编码。

	假如secretkey = d9f4aa7ea6d94faca62cd88a28fd5234，
	MD5之后的baseString为上一步生成的0829d4012497c14a30e7e72aeebe565e，
	则HmacSHA1加密之后再进行base64编码得到的signa为： IrrzsJeOFk1NGfJHW6SkHUoN9CU=
1
2
3

备注：

  • secretkey：在 应用管理>应用令牌管理中获取，请调用方注意保管；
  • signa的生成公式：HmacSHA1(MD5(appid + ts)，secretkey)

# sm4数据加密

接口地址 https://183.66.35.94:8089/encryptDataSM4
请求方式 POST
consumes ["application/json"]
produces ["/"]
接口描述 对输入的数据（形如密码、关键字、敏感信息等）进行加密。 在密钥管理对应加密机厂商下创建SM4数据加解密密钥。 获取密钥，在调用该api时作为参数传递。 注意：SM4数据加解密为同一密钥。
请求参数
参数名称 参数说明 请求类型 是否必须 数据类型
encryptData 待加密数据 query true String
secretKey sm4加密密钥 query true String
响应状态
状态码 说明
200 成功
500 失败
响应参数
参数名称 参数说明 参数类型
code 状态码 Integer(int32)
data 返回数据 Object(加密数据)
msg 提示信息 String
success 执行是否成功（true:成功 false:失败） Boolean
